# Publication Brief: Before the Agent Starts

## 1. Publication Title
DON’T GO BROKE IN THE AGE OF AI: Before the Agent Starts

## 2. Publication Type
Paid PDF Booklet

## 3. Series / Product Family
Don't Go Broke (Non-Platform Booklet Sequence)

## 4. Intended Reader
- First-time AI builders
- Solo founders
- Side-project creators
- Students
- Small business owners
- Non-technical builders using AI coding tools

## 5. Reader Problem
First-time AI builders often treat AI coding agents as advanced autocomplete, failing to realize that agents have the autonomy to modify files, run commands, and restructure projects. If unchecked, agents can silently expand scope, break existing code, and generate massive unreviewable PRs.

## 6. Core Promise
This booklet promises to provide a practical framework to set safe operating parameters and boundaries *before* turning an autonomous agent loose on a codebase.

## 7. Core Lesson
An AI coding agent is not harmless autocomplete; it is a fast worker that requires strict management. Builders must establish clear boundaries regarding what the agent can touch, the tools it can use, and when it must stop and ask for permission.

## 8. Mistake Prevented
It prevents the expensive and demoralizing mistake of an AI agent silently expanding its scope, breaking existing code, generating massive unreviewable changes, or heading down a complex, incorrect architectural path without oversight.

## 9. Reader Transformation
The reader shifts from a mindset of "let the AI build it" (passive observer) to "I manage the AI's boundaries" (active supervisor), taking control of the agent's scope and review cycle.

## 10. Scope Boundaries
The scope is strictly limited to the execution and management phase of working with an autonomous coding agent. It focuses on setting up rules, boundaries, and safe task sequencing.

## 11. What this publication is not
- It is not a general AI safety or ethics essay.
- It is not a prompt-engineering manual.
- It is not a coding tutorial.
- It does not focus on one specific platform (e.g., Devin, AutoGPT, Copilot Workspace).
- It is not a repeat of `Before the Build Starts`.

## 12. Relationship to `Before the Build Starts`
`Before the Build Starts` focuses on the product design and blueprinting phase. `Before the Agent Starts` assumes the builder already has that blueprint and is now ready to begin the execution phase using an autonomous agent.

## 13. Relationship to `Before You Trust the First Draft`
This booklet naturally sets up the next critical phase: how to rigorously test, review, and trust the actual code the agent produces (`Before You Trust the First Draft`), by ensuring the agent operates within testable increments.

## 14. Recommended Structure at a High Level
- **Introduction**: The difference between autocomplete and autonomous agents.
- **Defining Boundaries**: What the agent can and cannot change.
- **The Safe First Task**: How to assign small, verifiable tasks.
- **Review Cycles**: When the agent must stop and ask.
- **Scope Creep Defense**: How to avoid silent expansion.
- **Checklists**: Quick reference for setting up agent instructions.

## 15. Editorial Voice and Tone
- Practical, calm, and protective.
- Builder warning a builder.
- Not alarmist, not technical gatekeeping.
- Not anti-AI, not corporate SaaS.

## 16. Practical Examples or Scenarios
- Scenario: An agent rewrites the entire routing structure instead of adding one route.
- Example: A "Safe First Task" vs. an "Unbounded Task".
- Scenario: Setting explicit paths for the agent to modify (e.g., "only touch `src/components`").

## 17. Required Concepts to Cover
- What the agent may change vs. must not change.
- What files or folders the agent may touch.
- What tools the agent may use.
- What decisions the agent may not make silently.
- When the agent must stop and ask.
- What counts as a safe first task.
- How to review agent output before allowing the next step.
- How to avoid letting the agent expand scope silently.

## 18. Concepts to Avoid
- Specific syntax for one tool (like Cursor rules or specific Devin prompts), keeping it agnostic.
- General moral warnings about AGI.
- Complex architectural concepts irrelevant to boundary setting.

## 19. Success Criteria
A builder can read this booklet in under 30 minutes and immediately draft a solid "boundary document" or set of instructions to constrain their agent's first run.

## 20. Production Notes
- Preserve the visual style and length constraints established in `Before the Build Starts`.
- Keep diagrams simple and focused on workflow/boundaries.
- Do not begin drafting until the outline is approved.
