# Section Plan: Before the Agent Starts

## Field Note Opening
- **Purpose:** Establish the moment right before the builder lets the agent take its first action.
- **Takeaway:** Speed without direction is just faster chaos. Pause and set rules before starting.
- **Tone:** Calm and protective.

## 1. The Agent Is Not Just Typing Suggestions
- **Purpose:** Distinguish autocomplete/chat help from agentic action.
- **Takeaway:** You are managing a fast, eager worker, not just receiving syntax suggestions.
- **Tone:** Explanatory.

## 2. Decide What the Agent May Change
- **Purpose:** Define clear, explicit file/folder boundaries.
- **Takeaway:** Restrict the agent's playing field to only what is necessary for the immediate task.
- **Tone:** Practical and direct.

## 3. Decide What the Agent Must Not Touch
- **Purpose:** Introduce the concept of "do not touch" boundaries and protecting core assets.
- **Takeaway:** Explicitly protect your load-bearing working code.
- **Tone:** Grounding.

## 4. Define the Safe First Task
- **Purpose:** Teach readers to break down their blueprint into small, verifiable first steps.
- **Takeaway:** Start with a narrow, testable chunk of work, not a broad wish.
- **Tone:** Practical.

## 5. Set Tool and Command Boundaries
- **Purpose:** Address the serious risks of terminal/command execution by the agent.
- **Takeaway:** Explicitly forbid the agent from running destructive or heavy commands silently.
- **Tone:** Protective.

## 6. Decide What the Agent May Not Decide Alone
- **Purpose:** Prevent silent architectural decisions and scope creep.
- **Takeaway:** The agent should stop and ask for human input at major decision forks rather than guessing.
- **Tone:** Grounding.

## 7. Make the Agent Stop and Explain
- **Purpose:** Define stop points and approval gates.
- **Takeaway:** Dictate the workflow rhythm by requiring the agent to explain its plan before acting.
- **Tone:** Direct.

## 8. Review Before the Next Step
- **Purpose:** Explain the necessity of reviewing output before continuing.
- **Takeaway:** Defend against scope creep by verifying work before giving the next instruction.
- **Tone:** Instructive.

## 9. Boundary Checklist
- **Purpose:** Provide a practical, fill-in-the-blank tool for immediate use.
- **Takeaway:** A concrete mental check to complete before hitting "Enter".
- **Tone:** Practical.

## Field Note Closing
- **Purpose:** Reinforce the core lesson in a protective, encouraging way.
- **Takeaway:** Setting boundaries keeps you in the driver's seat.
- **Tone:** Empowering.
