# Booklet Production Double-Check Standard

This standard defines the mandatory double-checks and review gates required for every Don't Go Broke booklet before final PDF generation. No booklet may be published or compiled into a storefront-ready asset until all gates are cleared.

## 1. Manuscript Depth Gate
Before any layout proof generation begins, the reviewer must confirm:
- [ ] What is the exact word count?
- [ ] Does it meet the 5,500–7,000 word target?
- [ ] Is it safely above the 5,000-word minimum depth trigger?
- [ ] Does the manuscript contain real depth (e.g., worksheets, checklists, deep examples) rather than fluff or repetitive filler?
- [ ] Does this fundamentally feel like a comprehensive booklet, not just a brief or a pamphlet?

## 2. Density Gate
After generating a draft layout proof, the reviewer must confirm:
- [ ] What is the corrected-layout page count?
- [ ] What is the calculated words per page density?
- [ ] Is the density within the 190–220 target range? (Or if below 180, is it justified by being exceptionally worksheet-heavy?)
- [ ] Is the page count truly earned by the manuscript depth?
- [ ] Is the layout free of artificial inflation (i.e., the layout is not artificially padding the length)?

## 3. Layout Integrity Gate
The reviewer must inspect the physical rendering of the layout proof and confirm:
- [ ] No forced spacing or vertical padding was used merely to meet page targets.
- [ ] No oversized fonts were used.
- [ ] No excessive margins were applied.
- [ ] No forced section breaks were inserted that create artificial blank space.
- [ ] No awkward page breaks, orphans, or widows remain.
- [ ] No stranded headings at the bottom of pages.
- [ ] No bad splits in checklists, code blocks, or prompt blocks.

## 4. Reference Comparison Gate
The reviewer must compare the current booklet against the established benchmarks:
- [ ] How does it compare to `before-the-build-starts` (the gold-standard booklet)?
- [ ] How does its density compare to the existing platform pamphlets?
- [ ] **Core Question 1:** Is this manuscript clearly deeper than a pamphlet?
- [ ] **Core Question 2:** Does this feel like a real publication?
- [ ] **Core Question 3:** Is it comparable in quality and weight to the gold-standard booklet?

## 5. Human Visual Review Gate
Before final PDF generation, a human must visually open the proof PDF and explicitly approve:
- [ ] Front matter formatting
- [ ] Table of Contents (TOC) rendering and link accuracy
- [ ] Heading breaks and hierarchy
- [ ] Overall line spacing and readability
- [ ] List and checklist rendering
- [ ] Prompt and code block shading/rendering
- [ ] Field note blockquote rendering
- [ ] Overall visual consistency
- [ ] Final PDF readiness

*If a booklet fails any of these gates, it must be kicked back to the editorial or layout phase for correction. Final PDF generation is blocked until all boxes are checked.*
