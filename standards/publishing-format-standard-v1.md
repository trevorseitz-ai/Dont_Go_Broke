# Publication Brief Standard v1

## Purpose

This document defines the required handoff from a publishing project to Publishing Factory.

Publishing Factory should not require large custom writing prompts.

Publishing Factory should receive a Publication Brief and generate the required publication through its standard workflow.

This document defines the minimum information required.

---

# Core Principle

Publishing projects define:

* What should be created
* Why it should exist
* Who it serves

Publishing Factory determines:

* Research
* Outline
* Draft
* Editorial Review
* Final Copy

---

# Required Publication Brief Structure

## Publication Information

### Title

The publication title.

### Publication Type

One of:

* Brief
* Pamphlet
* Booklet
* Manual
* Book

Publication type must match the Publishing Format Standard.

### Series

Series name if applicable.

Examples:

* Don’t Go Broke
* AI Survival Manuals
* Decision Engine

### Version

Document version.

---

## Purpose

What should this publication accomplish?

One short paragraph.

---

## Audience

Who is this publication for?

Include:

* experience level
* target reader
* relevant constraints

---

## Core Lesson

The single most important thing the reader should remember.

One sentence.

---

## Required Concepts

List concepts that must appear.

These are outcomes, not wording requirements.

Example:

* Build with your eyes open
* Hidden costs are easier to avoid than recover from
* Small enough to understand is safer than impressive enough to show off

---

## Required Sections

List required sections.

Publishing Factory may improve structure but may not remove required sections without escalation.

---

## Source Material

List approved source material.

Examples:

* Research documents
* Existing pamphlets
* Field notes
* Previous publications
* Project standards

---

## Constraints

List hard requirements.

Examples:

* Platform-independent
* Beginner-safe
* No fear marketing
* No unsupported claims

---

# Publishing Factory Responsibilities

Upon receiving a valid Publication Brief, Publishing Factory must:

1. Create research.md
2. Create outline.md
3. Create draft.md
4. Run Publishing Room review
5. Create final.md
6. Return approval package

---

# Clarification Rule

If any of the following are missing:

* Publication Type
* Purpose
* Audience
* Core Lesson

Publishing Factory must stop and request clarification.

Publishing Factory must not guess.

---

# Ownership

Publishing projects own the Publication Brief.

Publishing Factory owns production.

Audience Factory owns promotion.
